3. AWS Shared Responsibility Model

AWS is responsible for security of the cloud. You are responsible for security in the cloud. The boundary shifts depending on the service type — click a row to understand what that means in practice.
Service type Example AWS is responsible for You are responsible for
IaaS EC2 Physical hardware, hypervisor, network infrastructure, host OS Guest OS patching, application security, firewall rules, IAM, data encryption
PaaS Lambda, RDS Hardware, OS, runtime patching, scaling, high availability Application code, IAM roles, data encryption, network controls
SaaS / Managed S3, DynamoDB, Cognito Infrastructure, durability, availability, service-level security Access control (bucket policies, IAM), encryption configuration, data classification
Global services IAM, CloudFront Global infrastructure, service availability Policy design, least privilege, MFA enforcement, all configuration

Click a row above to understand the responsibility boundary for that service type.

The key insight: As you move from IaaS to PaaS to managed services, AWS takes on more infrastructure responsibility — but you always retain responsibility for identity, access control, and data protection. No AWS service manages your IAM policies or encryption keys for you.

4. Compliance Frameworks

Compliance is not the same as security. Security is about reducing risk. Compliance is about demonstrating to a third party that you meet a defined standard. A system can be compliant but insecure, and secure but non-compliant. In practice, compliance frameworks provide a useful baseline of controls — but the goal is always genuine security, not just a checkbox.

Click a framework to expand it.

▶

HIPAA — Health Insurance Portability and Accountability Act

Who it applies to: Any organization handling Protected Health Information (PHI) — medical records, health data, insurance claims. Healthcare providers, insurers, and their business associates.

Key requirements: Access controls on PHI, audit logging of who accessed what, encryption of PHI at rest and in transit, breach notification within 72 hours.

Relevant AWS services:

CloudTrailKMSIAMCloudWatchMacie

In the Photo Viewer context: If photos contained medical imagery or patient data, HIPAA would require encryption at rest (KMS), audit logging of every access (CloudTrail data events), and strict access controls (IAM + Cognito).

▶

PCI DSS — Payment Card Industry Data Security Standard

Who it applies to: Any organization that stores, processes, or transmits cardholder data — payment processors, e-commerce sites, retailers.

Key requirements: Network segmentation to isolate cardholder data, no storage of full card numbers, encryption in transit (TLS 1.2+), quarterly vulnerability scans, annual penetration testing, MFA for all administrative access.

Relevant AWS services:

VPCWAFCloudTrailInspectorKMSSecurity Hub

In the Photo Viewer context: If the app processed payments for premium upgrades, PCI DSS would require the payment processing to be network-isolated in a private VPC subnet, TLS enforced via CloudFront, and quarterly scans via Inspector.

▶

ISO 27001 — Information Security Management System

Who it applies to: Any organization seeking to demonstrate a systematic approach to information security management. Commonly required in enterprise procurement and government contracts.

Key requirements: Risk assessment and treatment, security policies and procedures, access control policy, incident response plan, business continuity planning, regular internal audits.

Relevant AWS services:

AWS ConfigSecurity HubCloudTrailIAMOrganizations

In the Photo Viewer context: ISO 27001 would require formal access control policies (covered by IAM role model), continuous monitoring (Config + Security Hub), and documented incident response procedures (ASFF + IR report from Week 12).

▶

GDPR — General Data Protection Regulation

Who it applies to: Any organization processing personal data of EU residents, regardless of where the organization is based.

Key requirements: Lawful basis for processing personal data, right to access/erasure/portability, data minimization, privacy by design, breach notification within 72 hours, Data Protection Impact Assessment for high-risk processing.

Relevant AWS services:

MacieKMSCloudTrailS3Cognito

In the Photo Viewer context: Photos may contain faces — personal data under GDPR. Macie would detect PII in metadata. S3 bucket region selection matters for data residency. Cognito handles right-to-erasure for user accounts. CloudTrail provides the audit trail required for breach notification.

Compliance vs security: AWS can be compliant with all of these frameworks — and AWS holds certifications for many of them. But certification applies to AWS's infrastructure, not to your application. You inherit AWS's compliance posture for the infrastructure layer, but you must implement the controls for your application layer yourself.

5. AWS Well-Architected Framework

The Well-Architected Framework is AWS's set of design principles for building cloud systems. It has five pillars. Security is one of them — but good security also depends on the others. Click a pillar to expand it. The Security pillar is the focus of this course.

▶

🔐 Security — course focus

Protect information, systems, and assets

The security pillar covers how to protect data, systems, and assets while delivering business value through risk assessment and mitigation strategies.

Design principles:

  • Implement a strong identity foundation — least privilege, no long-term credentials
  • Enable traceability — audit logs for all actions and changes
  • Apply security at every layer — edge, VPC, compute, application, data
  • Automate security best practices — use policy as code
  • Protect data in transit and at rest
  • Keep people away from data — reduce direct access, use automation
  • Prepare for security events — incident response plans and simulations

This course covers every one of these principles through the Photo Viewer application build.

▶

⚙️ Operational Excellence

Run and monitor systems to deliver business value

Operational excellence focuses on running workloads effectively, gaining insights into operations, and continuously improving processes. Key practices include infrastructure as code, runbooks, and automated responses to operational events.

Relevant to this course: CloudTrail event logging, CloudWatch alarms and automated responses, Systems Manager for patch automation.

▶

🔄 Reliability

Recover from failures and meet demand

Reliability covers the ability of a workload to perform its intended function correctly and consistently, including the ability to recover from failures. Availability zones and multi-region deployments are the primary mechanisms.

Relevant to this course: S3 is automatically multi-AZ within a region. DynamoDB replicates across AZs. Understanding AZ vs regional vs global resilience — covered in Week 1.

▶

⚡ Performance Efficiency

Use resources efficiently at any scale

Performance efficiency focuses on using computing resources efficiently to meet system requirements, and maintaining that efficiency as demand changes and technologies evolve.

Relevant to this course: CloudFront edge caching reduces latency for users worldwide — one of the three problems with Stage 1 that CloudFront fixes in Week 3.

▶

💰 Cost Optimization

Avoid unnecessary costs

Cost optimization focuses on avoiding unnecessary costs by understanding spending over time and controlling fund allocation, selecting the right resource types and quantities, and scaling to meet business needs without overspending.

Relevant to this course: CloudWatch billing alarm and AWS Budgets — set up in Week 1 before any resources are created. Disabling CloudTrail data events after demos to avoid ongoing charges.

AWS added a sixth pillar in 2022: Sustainability — minimizing the environmental impact of cloud workloads. It covers selecting efficient regions, right-sizing resources, and preferring managed services over self-managed infrastructure.
